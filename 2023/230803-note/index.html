自定义测试弹幕

　　→效果如图，大概就是把测试服务器和样式生成器的里自动生成的随机文本替换成需要的内容，也是一种署名形式……
　　 这张截图居然刚好没有直接生成出我转世后的 ID 诶，乐【
　　另外还可以生成随机头像，不过将 ID 和头像对应大概还需要更复杂的代码，暂时就不做了。

　　测试文本存放在 blivechat/frontend/src/api/chat/ChatClientTest.js 中，按需要更改即可，注意每个逗号后面都需要添加空格或换行，否则 vue 会报错 :4o-2:
　　为测试弹幕添加随机头像需要将所需的图片存放在 blivechat/frontend/public/static/img 文件夹中，注意要放在 public 而不是 dist 里，否则 build 的时候会被自动清理掉【
　　为了测试大表情样式我添加了一些自定义表情，同一组图片也可以用作头像，所以随机头像直接引用和表情生成相同的代码就可以了。修改和测试文本同一个文件中的 EMOTICONS 数组，再将所有的 avatarUrl: avatar.DEFAULT_AVATAR_URL 查找替换为 avatarUrl: randomChoose(EMOTICONS) 即可。
　　另外修改了头像来源之后需要删掉第三行的 import * as avatar from './avatar'，不然 vue 也会报错 你又用不到这个东西导入过来干什么

　　哎呀这张图片怎么这么长我要编不下去了【草
　　我是不是应该给侧边排版的图片设置一个高度上限，这也太难排版了……

　　完成所有操作后依然需要在 blivechat/frontend 目录下执行 npm run build 并强制刷新浏览器，否则它会一直引用缓存中的 JS 脚本，我也不知道为啥，可能是 build 前端的时候没有清理缓存文件但是它既然能跑我们还是不要再纠结太多啦哈哈 :4o-3:

小插曲

　　今天突然一时兴起点开了评论后台，发现之前做着玩的 FF14 对话框弹幕样式居然真的有人在用，并且在重装丢失文件后通过浏览器缓存经过一系列失效的视频和动态找到了这里……
　　说不感动是假的，但又真的没那么感动了。
　　「被需要」和「被认可」的喜悦一旦涉及到 FF14 这个 tag 就会立刻被冲刷得黯然失色。到底还要多久才能和上辈子和解呢？我也不知道。
　　但至少这次的突然到访没有触发我的 PTSD，我可以心情平和地把 github 的仓库地址回复给 TA 再心平气和地补充一句「感谢喜欢」，或许也算可喜可贺，可喜可贺。
