纪念：电脑端初步建设完成

2021/09/02 字数统计： 1.4k 阅读时间： 2 mins 访问次数： ... 评论统计： ...
  • 维护日志
  • HKT-924

　　本来说好今天下午整一下那个 HUD 的动画效果的，结果我果然还是对 JS 有蜜汁望而生畏的恐惧感，于是改变计划决定先把分类页面的 css 写好，最后再修正一下手机上的预览效果？

警告：今日工作量已严重超过额定计划
　　　剩余电量低于 15%，请及时充电

　　我真是太看得起自己了……吃完午饭信誓旦旦说要搞什么手机 UI，结果 stylus 和 layout 相继出错搞得我心力交瘁。
　　layout 出了啥问题呢，分类的子页面和归档共用一个 post-list.ejs 作为主元素，但是不知道为什么一模一样的 layout 在渲染归档里的文章元素的时候就会少掉一个 </div> 闭合标签。然后原作者选择用 JS 逐个补充标签，然后到我这的结果就是用来做 css 样式的 list-body 断成了一截一截。
　　其实理论上我换个元素做 css 就能解决了，但是在发现这个问题之前我又先被 stylus 狠狠搞了一把心态。我不知道是我在一个类标签里塞的子元素样式太多了超过它的处理上限了还是怎样，我把每个括号手动缩进之后挨个检查确定全部闭合了，但是动画效果就是 tmd 做不出来，最后我还是选择另起一行用传统 css 的语法一条一条地写动画，实在是不想再从头挨个修改类名，最后是另外新建了一个单独的 layout 解决的。
　　但总之我是做出来了……类似尼尔存档的 UI 界面……还解决了给文章添加指定封面参数……
　　好家伙，打出这行字的时候我真是跟虚脱一样……

工作小结

　　那么本站基础 UI 已于地球时间 21.9.2 施工完毕，目前已实现的功能：

  • 两种文章列表形式
    • 卡片瀑布流
    • 仿尼尔存档 UI
      • 细节上还有亿点丑，但是一部分没什么思路另一部分改起来很快，先鸽置一下
  • 分类列表页面
    • 手写的【
  • 相册展示
    • 为文章单独指定了封面属性
    • 为封面添加了文章链接
  • 自定义域名
  • 长毛象认证
  • 书签形翻页器
  • RSS 订阅链接
  • 自定义浏览器图标
