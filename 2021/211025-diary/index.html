写作

　　写作方面可以说是一败涂地，不管是 Rinne 还是泠泺都兜兜转转回到原地。
　　一方面是写的东西目前还都不能公开，另一方面是我发现小时候那种「开始写作之前要先去文具店挑好顺眼的新本子和笔」的怪癖一直延续到今天都没有什么改善，甚至在掌握的知识变多了以后写作的准备条件也变得更加繁琐了。当年只要买了漂亮的新本子和笔就可以开开心心地开始写作，而如今甚至要自己动手部署一个笔记程序……
　　亲爱的 Obsidian，你的夜间模式和紫色高亮是那么貌美可爱，可是你为什么™不支持全角空格缩进，啊我知道全角空格和单行换行都不属于 markdown 标准格式但是，为什么我回车的时候你要自动给我加两个半角空格啊，我不需要这玩意好吗我谢谢你 :4o-2:
　　还有一个 Focalboard 一直躺在我的收藏夹里吃灰，如果不写这篇玩意我可能都要忘记它是什么了。看起来用本地部署 + github 同步存档的方式即使不需要 VPS 也可以实现远程协作，可是它没有中文文档，我真是痛苦面具。
　　最后还是要说一句：去死吧 Notion！

新玩具：造语

　　根据人格数据记载我曾经在小学的时候就整过这个玩意，当时还对着字典给每个字都创造新读音，但是本质上是做一个老师读不懂的密码系统所以在语法上没有什么特殊规划。
　　这次重新开始做人工语言初衷一方面是补全 OC 世界观，另一方面是想创造一种容易押韵写歌词的语言，所以它的基本语法是这样的：每个句式都有一个固定的助词，确保相同情感的完整句韵尾可以达成一致；同时也有简写的形式，防止所有句子都以同一个形式结尾。
　　以现有的词库举例：我 = mi，猫 = nemi，爱 = amos，肯定 = es/cones/Con..es；我爱猫猫 = Es mi nemi amos，完整句 = Con mi nemi amos es.
　　这么一看这个完整句词缀好像代码的花括号噢……那么不同的句式就是不同的函数，括号里是按 SOV 顺序录入的不同变量，理解句子的过程就是执行这个函数的过程，宇宙猫猫.jpg
　　不过这只是基本设计语法，因为使用了 SOV 的语序，所以主格和宾格需要用助词隔开，否者二者词性相同会产生混乱；另外形容词还没想好是前置还是后置；大部分语法大概还是会参考日文，因为这是我最熟悉的 SOV 语序的语言了，但是日文的助词又太多了，应该会进行一定程度的精简……
　　计划是先整理出音节表，为所有助词分配好单音后再进行进一步完善。我是不是可以在 ark 里开个新页面了？
